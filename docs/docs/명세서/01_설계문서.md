## 1. 프로젝트 개요

### 1.1 프로젝트 정보
| 항목 | 내용 |
|------|------|
| 프로젝트명 | Local OS Super Agent (Steer) |
| 버전 | 1.0 |
| 작성일 | 2026-02-18 |
| 대상 플랫폼 | macOS 12+ (Monterey), Windows 10+ |
| 라이선스 | MIT License |

### 1.2 프로젝트 목적
사용자의 컴퓨터 사용 패턴을 **자동 수집·분석**하여 반복 작업을 감지하고, **자동화 워크플로우를 추천**하며, **자연어 명령만으로 OS를 직접 제어**할 수 있는 로컬 에이전트 시스템이다.

### 1.3 핵심 철학
> **"LLM plans, Rust enforces, native layer executes"**

| 원칙 | 설명 |
|------|------|
| Execution over Chat | 단순 조언이 아닌 실제 OS 조작 수행 |
| Safety First | LLM은 OS를 직접 제어하지 않으며, 엄격한 Policy Engine을 통해서만 실행 |
| Closed Loop | 실행 → 검증 → 복구의 폐쇄 루프 구조 |
| Zero Trust | LLM 출력은 무조건 불신, 모든 액션은 보안 분류 후 실행 |

## 3. 문제 정의 및 해결 방안

### 3.1 문제 정의
- 직장인/개발자의 일일 컴퓨터 작업 중 **30~40%가 반복 작업**으로 구성됨
- 기존 RPA(로봇 프로세스 자동화) 도구는 **전문 지식 필요**, 높은 진입 장벽
- 매크로/스크립트 방식은 **환경 변화에 취약**, 유지보수 비용 높음

### 3.2 해결 방안
1. **자동 패턴 감지**: 백그라운드에서 사용 패턴을 수집하여 반복 작업을 AI가 자동 발견
2. **자연어 자동화**: "항공권 찾아줘" 같은 자연어 명령으로 복잡한 작업 수행
3. **보안 내장**: 5단계 보안 모델로 위험한 작업은 사용자 승인 후 실행

### 3.3 차별점
| 기존 RPA | Steer Agent |
|----------|-------------|
| 수동 워크플로우 설계 | AI 자동 패턴 감지 & 추천 |
| GUI 기반 시나리오 작성 | 자연어 명령 |
| 클라우드 의존 | 완전 로컬 실행 (개인정보 보호) |
| 단순 매크로 반복 | LLM 기반 지능형 판단 + 검증 루프 |
| 보안 모델 부재 | 5단계 보안 + Write Lock |

## 9. 보안 아키텍처

### 9.1 5단계 보안 모델

```
┌─────────────────────────────────────────┐
│  5단계: Privacy Guard (PII 마스킹)       │
│  ┌───────────────────────────────────┐  │
│  │  4단계: API Auth (Key 미들웨어)    │  │
│  │  ┌─────────────────────────────┐  │  │
│  │  │  3단계: Command Classifier  │  │  │
│  │  │  ┌───────────────────────┐  │  │  │
│  │  │  │  2단계: Policy Engine │  │  │  │
│  │  │  │  ┌─────────────────┐  │  │  │  │
│  │  │  │  │ 1단계: Tool     │  │  │  │  │
│  │  │  │  │ Policy (Allow/  │  │  │  │  │
│  │  │  │  │ Denylist)       │  │  │  │  │
│  │  │  │  └─────────────────┘  │  │  │  │
│  │  │  └───────────────────────┘  │  │  │
│  │  └─────────────────────────────┘  │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

| 단계 | 모듈 | 역할 |
|------|------|------|
| 1단계 | Tool Policy (`tool_policy.rs`) | 도구 카테고리별 Allow/Denylist |
| 2단계 | Policy Engine (`policy.rs`) | 액션을 Safe/Caution/Critical로 분류 |
| 3단계 | Command Classifier (`security.rs`) | 셸 명령 위험도 분류 (rm, sudo 등 차단) |
| 4단계 | API Auth (`api_server.rs`) | API Key 미들웨어 인증 |
| 5단계 | Privacy Guard (`privacy.rs`) | PII 마스킹 (이메일, 전화번호, 카드번호) |

### 9.2 Write Lock 메커니즘
| 항목 | 설명 |
|------|------|
| 기본 상태 | **LOCKED** (읽기 전용) |
| 해제 방법 | 사용자 명시적 승인 (CLI Confirm / UI Button) |
| 자동 잠금 | 마지막 액션 후 N초 경과 시 자동 잠금 |

### 9.3 액션 보안 등급
| 등급 | 예시 액션 | 처리 |
|------|-----------|------|
| **Safe** | `ui.find`, `screen.capture`, `app.list` | 자동 승인 |
| **Caution** | `ui.click`, `keyboard.type` | Write Lock 해제 필요 |
| **Critical** | `file.delete`, `cmd.exec`, `submit.payment` | 건별 사용자 승인 필수 |

### 9.4 아웃바운드 정책
- **메일 발송**: 단일 수신자 강제, 본문/제목 필수, 발송 확인
- **텔레그램**: 본문 필수, 최대 120K 문자, 대상 ID 허용/차단 목록
- **민감정보**: 카드번호, 주민번호 등 자동 입력 금지

## 14. 법적 준수 및 개인정보 보호

### 14.1 관련 법규

| 법규 | 적용 범위 | Steer의 준수 방안 |
|------|-----------|-------------------|

| **GDPR** (EU) | EU 사용자 개인정보 | 데이터 최소 수집, 삭제권(리텐션), 동의 기반 |
| **CCPA** (미국 캘리포니아) | 소비자 개인정보 | 수집 항목 고지, 삭제 요청 대응 |
| **정보통신망법** (한국) | 정보통신서비스 이용자 보호 | 통신 비밀 보호, 접근 권한 최소화 |

### 14.2 개인정보 보호 설계 원칙 (Privacy by Design)

| 원칙 | 구현 |
|------|------|
| **데이터 최소 수집** | Allowlist 27개 앱만 수집, Denylist 앱(비밀번호 관리자 등) 완전 차단 |
| **로컬 전용 처리** | 모든 데이터는 로컬 SQLite에만 저장. 외부 서버 전송 없음 |
| **PII 자동 마스킹** | 이메일, 전화번호, 12자리+ 숫자 자동 레닥션 (`privacy.rs`, `privacy.py`) |
| **URL 도메인만 보존** | 전체 URL 저장 안 함, 도메인만 보존 (`keep_domain_only: true`) |
| **자동 삭제** | events 7일, sessions 30일, routines 90일 자동 삭제 |
| **민감 키 드롭** | `content`, `body`, `clipboard` 등 민감 페이로드 키 저장 안 함 |
| **사용자 동의** | 첫 실행 시 수집 범위·목적 고지 및 동의 UI |

### 14.3 데이터 처리 흐름과 법적 근거

```
이벤트 발생
    │
    ▼
Denylist 앱? ─── Yes ──→ 이벤트 전체 DROP (저장 안 함)
    │ No
    ▼
Allowlist 앱? ─── No ──→ 이벤트 전체 DROP
    │ Yes
    ▼
PII 마스킹 (이메일, 전화번호, 카드번호)
    │
    ▼
URL → 도메인만 보존, title 64자 제한
    │
    ▼
민감 키(content, body, clipboard) DROP
    │
    ▼
로컬 SQLite 저장 (외부 전송 없음)
    │
    ▼
자동 리텐션 (7일 후 삭제)
```

### 14.4 OpenAI API와 데이터 관계

| 항목 | 설명 |
|------|------|
| OpenAI로 전송되는 데이터 | 사용자의 **자연어 명령 텍스트만** 전송 (예: "항공권 찾아줘") |
| 전송되지 않는 데이터 | 수집된 이벤트, 앱 사용 패턴, PII, 파일 내용 |
| OpenAI 데이터 정책 | API를 통해 전송된 데이터는 모델 학습에 사용되지 않음 (OpenAI API Terms) |

## 15. 경쟁 우위 지속 가능성 (기술 해자)

### 15.1 왜 경쟁사가 쉽게 따라할 수 없는가

| 해자 (Moat) | 설명 | 방어 기간 |
|-------------|------|-----------|
| **로컬 데이터 축적** | 사용자별 3개월+ 행동 패턴 데이터가 로컬에 축적. 경쟁사 진입 시 데이터 콜드 스타트 문제 | 장기 |
| **5단계 보안 아키텍처** | Rust로 구현된 Policy Engine + Write Lock + PII 마스킹. 보안을 사후 추가하는 것은 매우 어려움 | 장기 |
| **Rust 기반 코어** | 82개 모듈(111개 소스 파일, 2.0MB+)의 Rust 코드베이스. 같은 수준의 성능+안전성을 Python/JS로 구현 불가 | 중기 |
| **Closed Loop 실행 구조** | LLM Plan → Execute → Verify → Approve 폐쇄 루프. 단순 API 래퍼와 근본적 차이 | 중기 |
| **워크플로우 생태계** | 사용자 축적 루틴 → n8n 워크플로우 자동 생성. 사용할수록 가치 증가 (네트워크 효과) | 장기 |

### 15.2 경쟁사 대응 시나리오

| 시나리오 | 위협 수준 | 대응 |
|----------|----------|------|
| Apple이 Shortcuts에 AI 추가 | 높음 | Steer는 크로스앱 + 크로스플랫폼. Shortcuts는 단일 앱 내 자동화 한정 |
| OpenAI가 데스크톱 에이전트 출시 | 높음 | Steer는 완전 로컬 + 프라이버시. OpenAI는 클라우드 의존 |
| 기존 RPA 회사가 AI 추가 | 중 | RPA는 엔터프라이즈 중심. 개인/소기업 시장은 가격 + 사용성에서 Steer 우위 |

## 부록 A: 용어 사전 (Glossary)

| 용어 | 설명 |
|------|------|
| **Steer** | 프로젝트 코드명. "조종하다"의 의미로, 자연어로 OS를 조종한다는 비전 |
| **Shadow** | 백그라운드 데이터 수집 기능 (F01). 사용자 행동을 "그림자"처럼 추적 |
| **Routine** | 반복 감지된 사용자 행동 패턴. 3일 이상 감지 시 루틴 후보로 등록 |
| **Handoff** | 루틴 후보를 n8n 워크플로우로 변환하기 위한 패키징 단계 |
| **Intent** | 사용자 자연어 입력에서 추출한 의도 (FlightSearch, ShoppingCompare 등) |
| **Slot** | Intent 실행에 필요한 파라미터 (출발지, 도착지, 날짜 등) |
| **Plan** | LLM이 생성한 실행 단계 목록 (navigate → fill → click → extract) |
| **Write Lock** | 기본 LOCKED 상태에서 사용자 승인 시에만 OS 변경 허용하는 보안 메커니즘 |
| **Policy Engine** | 모든 액션을 Safe/Caution/Critical로 분류하는 보안 엔진 |
| **Approval Gate** | Critical 액션에 대해 사용자 명시적 승인을 요구하는 게이트 |
| **Completion Score** | 자연어 태스크 실행 완료도를 0~1 점수로 평가한 지표 |
| **SSOT** | Single Source of Truth. Rust 코어가 모든 로직의 유일한 진실 소스 |
| **n8n** | 오픈소스 워크플로우 자동화 플랫폼. Docker로 로컬 실행 |
| **WAL** | Write-Ahead Logging. SQLite의 동시 읽기/쓰기 최적화 모드 |
| **PII** | Personally Identifiable Information. 개인 식별 가능 정보 |

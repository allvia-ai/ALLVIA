# STEER OS (Allvia) 15분 테크 피칭덱 스크립트 (Tech-Focused Pitch Deck Script)

> **발표자 노트:** 본 스크립트는 투자자(VC), 테크 리더(CTO), 심사역 등을 대상으로 STEER OS의 '기술적 해자(Tech Moat)'와 '아키텍처 결정(ADR)'을 딥다이브하여 설명하는 15분 분량의 무대 발표용 대본입니다. 발표 시 슬라이드 전환 타이밍에 맞춰 구어체로 작성되었습니다.

## [Slide 1: 오프닝 & 문제 제기 (0:00 - 1:30)]
**"장난감이 아닌, 인프라를 만들다."**

"안녕하십니까. 저는 기업의 반복 노동 운영 체제를 근본부터 다시 쓰는 'Allvia(구 STEER OS)' 팀의 발표자입니다.

최근 1년 사이 수많은 AI 에이전트들이 쏟아져 나왔습니다. 클로드 컴퓨터 유즈부터 시작해 깃허브에서 10만 스타를 받은 오픈클로(OpenClaw)까지 말이죠. 이들은 데모 영상에서는 마법처럼 움직입니다. 하지만 왜 정작 은행이나, 대기업 재무팀, 핵심 오퍼레이션 부서에는 단 한 대도 도입되지 못하고 있을까요?

이유는 단순하고 치명적입니다. **'책임질 수 없기 때문'**입니다.
기존 에이전트들은 화면이 조금만 바뀌어도 빙글빙글 헤매고(Hallucination), 그 상태에서 멋대로 거래처에 이메일을 보내거나 기밀 데이터를 삭제해 버릴 수 있는 '통제 불가능한 시한폭탄'입니다.

저희 팀은 'AI가 얼마나 똑똑한가'를 경쟁하지 않습니다. 대신 **'AI가 미쳤을 때(사고를 칠 때) 어떻게 하드웨어적으로 브레이크를 걸고 인간의 통제망 안에 가둘 것인가'**에 모든 엔지니어링 역량을 쏟아부었습니다. 

오늘은 저희가 왜 유행하는 기술 스택을 모두 버리고, 이토록 무식하고 단단한 'Zero-Trust OS Agent 아키텍처'를 밑바닥부터 구축했는지, 그 기술적 결정(Tech Stack Decision)들을 말씀드리겠습니다."

## [Slide 2: 백엔드/프론트엔드 - 크롬(Electron)을 버리고 Rust와 Tauri를 선택하다 (1:30 - 4:00)]
**"점유율 30MB의 극한 경량화와 OS 네이티브 제어"**

"B2B 환경에서 고객사 PC에 백그라운드로 하루 종일 떠 있어야 하는 에이전트의 제1 원칙은 '가벼움'입니다.

시장에 나온 99%의 데스크탑 앱들은 개발이 편하다는 이유로 크롬 브라우저 엔진을 통째로 내장하는 '일렉트론(Electron)' 기반으로 만들어집니다. 당연히 켜기만 해도 메모리를 200~300MB씩 잡아먹습니다. 보안이 철저하고 리소스가 빠듯한 기업 환경에서는 설치부터 강력한 저항에 부딪힙니다.

그래서 저희는 과감히 일렉트론을 버렸습니다. 대신 **메모리 안전성과 C/C++ 급의 퍼포먼스를 내는 러스트(Rust) 언어와 초경량 래퍼인 타우리(Tauri) 프레임워크**를 채택했습니다. 

그 결과는 압도적입니다. 앱 배포 용량은 10MB 미만이 되었고, 백그라운드 대기 메모리 점유율을 무려 85%나 깎아내어 **30MB 수준**으로 만들었습니다. 사용자들은 저희 에이전트가 켜져 있는지도 모를 만큼 PC가 쾌적합니다. 프론트엔드 역시 무거운 프레임워크를 덜어내고 바닐라 JS와 Tailwind CSS 조합으로 컴파일 타임 렌더링 수준의 극단적인 반응성을 챙겼습니다.

무엇보다 Rust의 도입은 단순히 메모리 때문만이 아닙니다. 에이전트가 마우스를 움직이려면 OS 커널 가장 밑바닥의 API에 접근해야 합니다. 저희는 Rust의 강력한 비동기 런타임인 Tokio 엔진을 기반으로 macOS의 네이티브 바인딩을 직접 제어하여, 중간 딜레이를 완전히 삭제했습니다."

## [Slide 3: 인식망(Perception) - 비전(Vision) API의 허상을 찢다 (4:00 - 6:30)]
**"픽셀 단위 사진이 아닌, OS UI 뼈대를 엑스레이로 직독직해(Direct Reading)하다"**

"다음은 에이전트가 화면을 인식하는 방식입니다.

클로드 컴퓨터 유즈를 포함한 대부분의 시각 기반 VLM 에이전트들은 화면을 '스크린샷'으로 찍어 픽셀 쪼가리로 분석합니다. 이건 매우 멍청하고 값비싼 방식입니다. 버튼이 10px만 움직여도 헛손질을 하고, API를 한 번 쏠 때마다 엄청난 레이턴시(지연)와 토큰 비용을 태워버립니다.

저희의 접근은 완전히 다릅니다. 저희는 모니터 겉면의 사진을 찍지 않습니다. 대신 운영체제(OS) 깊숙한 곳의 뼈대, 즉 **'접근성 트리(Accessibility API)'를 엑스레이처럼 직접 스캔**합니다. 구조를 직독직해하기 때문에 해상도가 달라지든 테마가 바뀌든 100% 명중시킵니다. API 레이턴시 없이 0.1초 만에 타겟을 찾아내죠.

행동(Action) 단계에서도 마찬가지입니다. 로봇이 물리적으로 마우스를 핑퐁핑퐁 움직이면, 중간에 사용자가 마우스를 건드리는 순간 워크플로우 전체가 꼬여버립니다. 저희는 무식한 마우스 트래킹을 버렸습니다. 화면 구조를 파악한 즉시, **운영체제 내부 시스템 콜(System Call)을 통해 진짜 클릭 이벤트를 하드웨어 단에서 쏴줍니다(Synthesize).** 즉, 겉으로는 커서가 안 움직여도 백그라운드에서 완벽히 코어 이벤트가 실행되며, 인간과 AI의 동선 충돌조차 차단해 버린 것입니다."

## [Slide 4: 보안/통제 인프라 - 가장 안전한 "사전 결재망 (Approval-Before-Execute)" (6:30 - 10:00)]
**"일단 저지르고 나중에 수습하는게 아니라, 위험한 행동은 계획 단계에서 원천 봉쇄합니다"**

"여기까지는 속도와 정확성이었다면, 이제부터가 저희가 진짜 B2B 엔터프라이즈의 표준이 될 수밖에 없는 **보안과 통제의 기술(Risk Classification & Approval Gate)**입니다.

일반적인 오픈소스 에이전트들은 코드가 허술합니다. '일단 이메일을 발송(Execute)'해 버린 뒤에, 어랏 잘못 보냈네? 하고 '나중에 검증(Verify)'합니다. 이건 엔터프라이즈 보안 설계상 매우 치명적입니다. 대형 컴플라이언스 위반 사고가 터지죠.

저희의 실행 파이프라인은 완벽한 선후 관계를 가집니다. **[의도파악] -> [계획(Plan)] -> [위험도 분류(Risk Classification)] -> [사전 결재망(Approval Gate)] -> [실행(Execute)] -> [검증(Verify)]** 순서입니다.

AI가 화면을 단순히 읽고 분류하는 건 자유롭게 둡니다. 하지만 'Plan' 단계에서 키보드를 쳐서 내용을 바꾸거나, 돈을 송금하거나, 이메일 전송 버튼을 누르는 등 **'치명적이고 돌이킬 수 없는 동작(Critical Action)'**이 계획되는 순간! 결코 실행(Execute) 단으로 코드가 넘어가지 않습니다.

스레드(Thread)가 그 즉시 정지되고, 사용자의 모니터 한가운데 강제로 팝업이 하나 뜹니다.
*"[Allvia 결재 요청] 현재 기밀 문서가 첨부된 이메일을 외부로 전송할 계획입니다. 실행을 승인하시겠습니까?"* 

네, 이렇게 **클릭하기 '전'에 승인을 받는 사전 결재망(Pre-Execution Approval Gate)**입니다. 인간 담당자가 눈으로 확인하고 '승인(Approve)' 버튼을 누르기 전까지 에이전트는 OS에 단 한 줄의 이벤트도 쏘지 못합니다. 

게다가 망 장애로 에이전트가 "전송" 명령을 실수로 두 번 보내는 중복 사고를 막기 위해, 코어단에 **멱등성(Idempotency) 설계**를 적용했습니다. 어떤 장애가 와도 한 번 실행된 동작은 OS 단에서 절대 중복 실행되지 않습니다. 수집된 통제 로그 또한 철저히 마스킹되어 **로컬 기기의 암호화 DB(Zero-Knowledge 스토리지)**에 고립 보관됩니다."

## [Slide 5: 신뢰성을 위한 운영(Ops) 설계 - Reconcile과 Release Gate (10:00 - 13:00)]
**"무책임한 데모를 넘어, 실패 복구가 가능한 살아있는 오퍼레이션"**

"실제 기업 환경의 데스크탑은 인터넷이 끊기고, 예상치 못한 팝업이 뜨는 최악의 진흙탕입니다. 단순히 '명령어 실행기'로는 하루도 버티지 못합니다.

저희 에이전트의 두뇌는 **상태 머신(State Machine)** 기반으로 정교하게 짜여 있습니다. 행동 중간에 네트워크가 끊어지면 지수 백오프(Exponential Back-off)로 재시도하고, 영영 끊어지면 무턱대고 뻗어버리는 게 아니라 **작업을 수행하기 직전의 안전한 상태로 롤백(Reconcile)** 시켜놓습니다.

에이전트 프로그램 자체의 신뢰성 검증도 극단적입니다. 개발팀 내에 500개 이상의 실제 고객사 성공 워크플로우(Baseline) 세트가 구축되어 있습니다. 코어 엔진을 단 한 줄이라도 수정해서 배포하려면(CI/CD), 이 500개의 테스트를 전부 통과해야 합니다. 단 하나라도 기능을 망가뜨리는 **퇴행(Regression)이 나타난다면 배포 파이프라인(Release Gate)이 강제로 닫힙니다.** 

우회 경로, 백도어, 디버깅 플래그요? 상용 릴리즈 빌드를 뽑을 때 컴파일러 단에서 해당 코드들을 아예 증발시키도록 설정되어 있어 제로 데이 해킹 가능성을 원천적으로 박살 냈습니다."

## [Slide 6: 클로징 - 수작업의 본질을 파고드는 기술 우위 (13:00 - 15:00)]
**"가장 위험한 일을, 가장 안전하게 위임하는 자율 인프라"**

"마무리하겠습니다.
범용 빅테크와 수많은 오픈소스 에이전트들은 계속해서 거대한 언어 모델의 지능(Intelligence)만 자랑하고 있습니다. 화려한 시를 쓰고 그림을 그리는 데는 그들이 유리할 겁니다.

하지만 엔터프라이즈의 돈은 '누가 기안서를 더 예쁘게 쓰느냐'가 아니라 **'누가 사고를 안 치고 이 지루한 운영(Ops) 정산 업무를 끝내주느냐'**에 있습니다.

저희 STEER OS(Allvia)는 
첫째, 30MB의 경량 아키텍처(Rust/Tauri)로 고객 도입 저항을 없앴고,
둘째, 물리 마우스를 버리고 OS 네이티브 뼈대를 파싱해 100%의 정확도를 잡았으며,
셋째, 치명적 사고 전(Pre-Execution)에 인간의 허락을 거치는 '사전 승인망(Approval Gate)'을 하드코딩한 **유일한 B2B 실전형 에이전트**입니다. 거기에 더해 최근 발표된 GPT-5.3 Spark 모델의 15배 빠른 생성 속도와 저희만의 '초경량 150토큰 프롬프트 설계'를 결합해 **단돈 10원도 안 되는 비용으로 0.1초 만에 실시간 OS 제어**를 달성했습니다.

저희는 이 철옹성 같은 기술 인프라를 바탕으로 순매출유지율(NRR) 125%에 도전하며 거대한 B2B 오퍼레이션 혁명을 증명하겠습니다.

감사합니다."
